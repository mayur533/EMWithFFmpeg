# Google Play Store Submission Requirements

## Complete Checklist for App Publishing

---

## 1. APK/AAB Requirements

### App Bundle (Recommended)
- **Format**: Android App Bundle (.aab)
- **Size**: Maximum 150 MB
- **Target API Level**: Must target API level 33 (Android 13) or higher
- **Signing**: Must be signed with your app signing key

### APK (Alternative)
- **Format**: Android Package (.apk)
- **Size**: Maximum 100 MB per APK
- **Multiple APKs**: Can upload multiple APKs for different configurations

---

## 2. App Icon Requirements

### Launcher Icon
- **Size**: 512 x 512 pixels
- **Format**: PNG (32-bit with alpha)
- **File Size**: Maximum 1 MB
- **Design**: Must be square with no rounded corners (Android handles this automatically)
- **Content**: No alpha/transparency in outer 10% of icon

### Notification Icon
- **Size**: 24 x 24 dp
- **Format**: PNG with transparency
- **Color**: White icon on transparent background

---

## 3. Screenshot Requirements

### Phone Screenshots (REQUIRED)
- **Quantity**: Minimum 2, Maximum 8
- **Dimensions**: 
  - Minimum: 320 pixels on short edge
  - Maximum: 3840 pixels on longest edge
  - Aspect Ratio: 16:9 or 9:16 (portrait or landscape)
- **Recommended Sizes**:
  - Portrait: 1080 x 1920 pixels (9:16)
  - Portrait: 1080 x 2340 pixels (9:19.5) for notched devices
  - Landscape: 1920 x 1080 pixels (16:9)
- **Format**: PNG or JPEG
- **File Size**: Maximum 8 MB per screenshot
- **Content**: Must show actual app functionality, no mockups

### 7-inch Tablet Screenshots (OPTIONAL but recommended)
- **Quantity**: Minimum 2, Maximum 8
- **Recommended Size**: 1080 x 1920 pixels or 1920 x 1080 pixels
- **Same requirements as phone screenshots**

### 10-inch Tablet Screenshots (OPTIONAL but recommended)
- **Quantity**: Minimum 2, Maximum 8
- **Recommended Size**: 1200 x 1920 pixels or 1920 x 1200 pixels
- **Same requirements as phone screenshots**

### Wear OS Screenshots (if applicable)
- **Quantity**: Minimum 1, Maximum 8
- **Size**: 384 x 384 pixels
- **Shape**: Square format

### Android TV Screenshots (if applicable)
- **Quantity**: Minimum 1, Maximum 8
- **Size**: 1920 x 1080 pixels
- **Orientation**: Landscape only

---

## 4. Feature Graphic

### Requirements
- **Size**: 1024 x 500 pixels (EXACT dimensions)
- **Format**: PNG or JPEG
- **File Size**: Maximum 1 MB
- **Content**: 
  - Must represent your app
  - No text overlays recommended (may be used in different contexts)
  - Should look good with your app icon overlaid
- **Usage**: Featured in Play Store, promotional materials
- **Status**: REQUIRED for featured placement

---

## 5. Promotional Graphics (Optional)

### Promo Video
- **URL**: YouTube video URL
- **Length**: 30 seconds to 2 minutes recommended
- **Content**: Showcase app features and UI

### TV Banner (for Android TV apps)
- **Size**: 1280 x 720 pixels
- **Format**: PNG or JPEG

---

## 6. App Store Listing Details

### Basic Information
- ✅ **App Name**: Maximum 50 characters
- ✅ **Short Description**: Maximum 80 characters
- ✅ **Full Description**: Maximum 4000 characters
- ✅ **App Category**: Choose primary category
- ✅ **Contact Email**: Must be valid
- ✅ **Website**: Optional but recommended
- ✅ **Phone**: Optional
- ✅ **Privacy Policy URL**: REQUIRED if app handles personal/sensitive data

### Language & Localization
- Default language: English (US)
- Add translations for other markets if applicable

---

## 7. Content Rating

### Age Rating Questionnaire
- Complete IARC questionnaire
- Answer questions about:
  - Violence
  - Sexual content
  - Language
  - Drugs/alcohol
  - User interaction
  - Data sharing
  - Location sharing

---

## 8. Privacy & Security

### Privacy Policy (REQUIRED)
- ✅ Must have a valid Privacy Policy URL
- ✅ Must explain data collection and usage
- ✅ Must be accessible and in-app (if collecting data)

### Data Safety Section (REQUIRED)
- Declare what data your app collects
- Explain how data is used
- Specify security practices
- Categories:
  - Location data
  - Personal information
  - Financial information
  - Photos and videos
  - Audio files
  - Files and documents
  - Calendar
  - Contacts
  - App activity
  - Device identifiers

### App Permissions
- Justify all permissions in the declaration form
- Remove unnecessary permissions

---

## 9. Target Audience & Content

- ✅ Select target age groups
- ✅ Declare if app is primarily for children
- ✅ Ads declaration (contains ads or not)
- ✅ In-app purchases declaration

## 10. Pricing & Distribution

### Pricing
- Free or Paid (cannot change from paid to free later)
- If paid: Set price in USD (auto-converts to other currencies)

### Distribution Countries
- Select countries/regions where app will be available
- Consider local laws and regulations

### Device Categories
- Phone
- Tablet
- Wear OS
- Android TV
- Android Auto
- Chromebook

## 11. App Release Types

### Internal Testing
- Up to 100 testers
- Releases available within minutes
- Full Google Play features

### Closed Testing (Alpha/Beta)
- Up to 1000+ testers
- Email list or Google Groups
- Collect feedback before production

### Open Testing
- Anyone can join
- No limit on testers
- Appears in Play Store as "early access"

### Production
- Publicly available
- Full release to all users
- Requires all assets and information

## 12. Technical Requirements

### App Signing
- ✅ Google Play App Signing (recommended)
- ✅ Upload key and app signing key managed by Google
- ✅ Keep your upload key keystore safe

### Version Information
- ✅ Version Name: User-facing (e.g., "1.0.0")
- ✅ Version Code: Internal integer (must increment)

### Android Version Support
- ✅ Minimum SDK: Should support at least 2-3 previous Android versions
- ✅ Target SDK: Must target API 33+ (Android 13)

## Screenshot Best Practices

### Content Guidelines
1. **Show Key Features**: Highlight main functionality
2. **User Flow**: Show logical progression
3. **Clean UI**: Remove personal data, use dummy content
4. **High Quality**: No pixelation or compression artifacts
5. **Status Bar**: Clean or hide it
6. **Captions**: Consider adding text overlays explaining features
7. **Consistency**: Use same device frame for all screenshots (optional)
8. **First Screenshot**: Most important, shows in search results

### Recommended Screenshot Order
1. **Main/Home Screen**: First impression
2. **Key Feature 1**: Most important functionality
3. **Key Feature 2**: Second most important functionality
4. **User Profile/Settings**: User customization
5. **Additional Features**: Other notable features
6. **Premium Features**: If applicable

### Tools for Screenshots
- **Android Studio**: Emulator screenshot tool
- **ADB**: Command-line screenshots from real device
- **Third-party tools**: MockUPhone, Previewed, AppMockUp
- **Frame generators**: Device Art Generator, Screener

❌ Missing required assets (screenshots, icon)
❌ Inappropriate content in screenshots
❌ Misleading app description
❌ Missing or invalid Privacy Policy
❌ Incomplete Data Safety section
❌ Trademark violations
❌ Crashes on launch
❌ Broken core functionality
❌ Not targeting latest API level
❌ Permissions not justified
❌ Intellectual property violations
❌ Deceptive behavior or malware
❌ User-generated content without moderation
